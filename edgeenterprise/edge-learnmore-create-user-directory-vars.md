<span data-ttu-id="7299b-107">Политики для изменения путей к каталогам данных (например, для настройки переменных поддержки [UserDataDir](microsoft-edge-policies.md#userdatadir) или [DownloadDirectory](microsoft-edge-policies.md#downloaddirectory).)</span><span class="sxs-lookup"><span data-stu-id="7299b-107">Policies for modifying data directory paths (For example, configuring the [UserDataDir](microsoft-edge-policies.md#userdatadir) or [DownloadDirectory](microsoft-edge-policies.md#downloaddirectory) support variables.</span></span> <span data-ttu-id="7299b-108">При настройке этих политик можно использовать переменные вместо жестко заданных путей.</span><span class="sxs-lookup"><span data-stu-id="7299b-108">When configuring these policies, you can use variables instead of hard-coded paths.</span></span> <span data-ttu-id="7299b-109">Например, для хранения данных профиля в разделе "Локальные данные приложения пользователя" в Windows вместо расположения по умолчанию.</span><span class="sxs-lookup"><span data-stu-id="7299b-109">For example, to store your profile data under user local application data on Windows instead of the default location.</span></span> <span data-ttu-id="7299b-110">Задайте политике [UserDataDir](microsoft-edge-policies.md#userdatadir) значение **${local_app_data}\Edge\Profile**.</span><span class="sxs-lookup"><span data-stu-id="7299b-110">Set the [UserDataDir](microsoft-edge-policies.md#userdatadir) policy to **${local_app_data}\Edge\Profile**.</span></span> <span data-ttu-id="7299b-111">В большинстве установок Windows 10 этот путь соответствует *C:\Users\\&lt;Current-user&gt;\AppData\Local\Microsoft\Edge\Profile*.</span><span class="sxs-lookup"><span data-stu-id="7299b-111">On most Windows 10 installations, this path resolves to *C:\Users\\&lt;Current-user&gt;\AppData\Local\Microsoft\Edge\Profile*.</span></span>

### <span data-ttu-id="7299b-116">Руководство по использованию переменных пути</span><span class="sxs-lookup"><span data-stu-id="7299b-116">Guidance for using path variables</span></span>

<span data-ttu-id="7299b-117">Перед использованием переменных для путей ознакомьтесь со следующими рекомендациями.</span><span class="sxs-lookup"><span data-stu-id="7299b-117">Review the following guidance before using variables for paths.</span></span>

- <span data-ttu-id="7299b-118">Все политики, включающие пути к расположениям, в которых Microsoft Edge хранит разные данные, зависят от платформы.</span><span class="sxs-lookup"><span data-stu-id="7299b-118">All policies that involve paths where Microsoft Edge stores different data are platform dependent.</span></span> <span data-ttu-id="7299b-119">Некоторые из этих политик доступны только на определенных платформах, тогда как другие можно использовать на всех платформах.</span><span class="sxs-lookup"><span data-stu-id="7299b-119">Some of these policies are available only on specific platforms, but others can be used on all platforms.</span></span>
- <span data-ttu-id="7299b-120">Во избежание ошибок, вызываемых приложениями, которые запускаются из разных расположений в различных условиях, проверьте, что пути являются абсолютными.</span><span class="sxs-lookup"><span data-stu-id="7299b-120">To avoid errors caused by applications starting from different locations on different occasions, make sure that paths are absolute.</span></span>
- <span data-ttu-id="7299b-121">Каждая переменная может быть использована в пути только один раз.</span><span class="sxs-lookup"><span data-stu-id="7299b-121">Every variable can occur only once in a path.</span></span> <span data-ttu-id="7299b-122">Для большинства из них это единственный эффективный способ использовать переменные, так как они разрешаются в абсолютные пути.</span><span class="sxs-lookup"><span data-stu-id="7299b-122">For most of them, this is the only meaningful way to use variables, because they resolve to absolute paths.</span></span>
- <span data-ttu-id="7299b-123">Почти все политики будут создавать путь, если его нет (по возможности в существующих обстоятельствах).</span><span class="sxs-lookup"><span data-stu-id="7299b-123">Almost all policies will create the path if it doesn't exist (if possible in the existing circumstances).</span></span>
